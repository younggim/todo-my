# AI Uplift — Spec

## 개요

사용자가 자신의 AI 활용 수준(현재·목표)을 입력하고 1일/1주/1개월 단위 todo를 관리하면서, 활용율 격차에 맞춘 tips와 큐레이션된 활용 사례, 매주 새로운 시도 1건 추천을 통해 AI 활용 행동을 늘려가는 single-user 웹 앱.

## 범위

### 포함
- **페이지 1 — 입력 & Tips** (Function 1 + Function 2)
  - 현재 AI 활용율과 목표 활용율 입력 (0–100)
  - 1일 / 1주 / 1개월 todo 추가·완료·삭제
  - 활용율 격차에 따른 시스템 추천 tips 자동 표시
  - 사용자 자작 tips 추가·편집·삭제
- **페이지 2 — AI 활용 사례** (Function 3)
  - 큐레이션된 AI 활용 사례 카드 목록
  - 외부 링크 새 탭 열기
  - "내가 시도함" 체크박스로 활용 표시 및 누적 카운트
- **페이지 3 — 시도 회고 & 다음 시도** (idea.md 핵심 루프)
  - 직군 선택 → 직군 매핑 시드에서 "다음 주 시도할 한 가지" 카드 1건 제시
  - 이번 주 시도 1–3줄 입력 (최대 5개)
  - 최근 4주 시도 회고 뷰 (지난 주 read-only)
- 페이지 간 명시적 네비게이션
- 모든 데이터의 재방문 시 자동 복원

### 제외
- **계정·로그인·다중 사용자** — 단일 사용자 모드. 협업 기능은 v1
- **LLM 자동 추천 / 자동 tips 생성** — 정적 시드로 시작. LLM은 retention 검증 후 v1
- **알림·푸시·이메일** — 회고 시점 알림은 v1
- **데이터 export/import / 백업** — v1
- **활용율 추이 그래프** — 시간에 따른 변화 시각화는 v1
- **모바일 전용 흐름** — 360px+에서 동작은 보장하나 별도 모바일 UX는 만들지 않음

## 시나리오

### 1. 첫 진입 — 빈 상태 표시
- **Given** — 처음 방문이라 저장된 데이터가 없음
- **When** — 페이지 1을 연다
- **Then** — 현재/목표 활용율 입력 필드가 빈 상태로 보이고, 1일/1주/1개월 todo 영역에는 "할 일이 없습니다" 안내가 표시되며, 페이지 1/2/3 네비게이션이 보인다

성공 기준:
- [ ] 페이지 1 진입 → "현재 활용율" 필드, "목표 활용율" 필드, 3개 todo 빈 영역, 페이지 네비게이션이 모두 보인다
- [ ] 페이지 2 진입 → 큐레이션된 사례 카드가 8개 이상 보인다 (체크 미반영 상태)
- [ ] 페이지 3 진입 → "직군을 선택하세요" 안내가 보이고, 회고 영역에는 "아직 기록된 시도가 없습니다" 안내가 보인다

### 2. 활용율 입력 → 격차 표시
- **Given** — 페이지 1, 활용율 미입력
- **When** — 현재 활용율 30, 목표 활용율 70 입력 후 저장
- **Then** — 입력값이 그대로 표시되고 "격차 40" 표시가 노출된다. 새로 고침해도 유지

성공 기준:
- [ ] 현재=30, 목표=70 입력 → "현재 30%", "목표 70%", "격차 40%" 텍스트가 화면에 보인다
- [ ] 새로 고침 → 같은 값과 격차가 유지된다
- [ ] 음수, 101 이상, 비숫자 입력 → "0과 100 사이 정수를 입력하세요" 에러 메시지가 해당 필드 아래 보이고 저장되지 않는다

### 3. 1일 Todo 추가 → 완료 → 삭제
- **Given** — 페이지 1, 1일 Todo 영역 빈 상태
- **When** — "ChatGPT로 회의록 요약하기" 입력 후 추가, 항목 체크, 항목 삭제
- **Then** — 추가 시 텍스트가 리스트에 보이고, 체크 시 취소선이 적용되며, 삭제 시 사라지고 새로 고침해도 유지된다

성공 기준:
- [ ] "ChatGPT로 회의록 요약하기" 추가 → 1일 Todo 리스트에 해당 텍스트 1개 항목이 보인다
- [ ] 항목 체크 → 텍스트에 취소선이 적용된다
- [ ] 삭제 클릭 → 리스트에서 사라지고, 새로 고침 후에도 보이지 않는다
- [ ] 빈 제목으로 추가 시도 → "내용을 입력하세요" 안내, 추가되지 않음

### 4. Weekly·Monthly Todo는 영역이 분리된다
- **Given** — 페이지 1, 1주 / 1개월 Todo 영역이 빈 상태
- **When** — 1주 영역에 "주간 회고를 AI로 작성", 1개월 영역에 "팀 워크플로 자동화 1건 도입" 입력
- **Then** — 각 텍스트가 자기 영역에만 보이고 다른 영역에 섞이지 않는다

성공 기준:
- [ ] 1주 영역 입력 → 1주 영역에만 항목이 보인다 (1일/1개월 영역엔 보이지 않음)
- [ ] 1개월 영역 입력 → 1개월 영역에만 항목이 보인다
- [ ] 각 영역의 추가·체크·삭제 동작이 1일 Todo와 동일하게 동작한다

### 5. 활용율 격차에 따른 시스템 Tips 자동 표시
- **Given** — 현재 활용율 30, 목표 70 (격차 40 = "큼" 카테고리)
- **When** — 페이지 1 하단의 "추천 Tips" 영역을 본다
- **Then** — "격차 큼" 카테고리의 시스템 tips가 3개 이상 표시된다. 격차가 다른 카테고리로 이동하면 표시되는 tips도 바뀐다

성공 기준:
- [ ] 격차 ≥ 30 (큼) → "큼" 카테고리 tips 3개 이상 표시
- [ ] 격차 10–29 (중간) → "중간" 카테고리 tips 3개 이상 표시
- [ ] 격차 ≤ 9 (작음) → "작음" 카테고리 tips 3개 이상 표시
- [ ] 활용율 미입력 → "활용율을 먼저 입력하면 추천 Tips가 보입니다" 안내가 표시되고, 시스템 tips는 보이지 않는다

### 6. 내 Tips 추가·편집·삭제
- **Given** — 페이지 1, "내 Tips" 영역이 빈 상태
- **When** — "코드 리뷰는 항상 AI에 1차 검토 시킨다" 추가 → 편집 → 삭제
- **Then** — 시스템 tips와 분리된 영역에 표시되고, 편집/삭제가 즉시 반영되며, 새로 고침에도 유지된다

성공 기준:
- [ ] "코드 리뷰는 항상 AI에 1차 검토 시킨다" 추가 → "내 Tips" 영역에 해당 텍스트가 보인다 (시스템 tips 영역과 시각적으로 분리됨)
- [ ] 편집 클릭 → 입력 필드로 전환, 수정 후 저장 시 새 텍스트가 보인다
- [ ] 삭제 → 해당 항목만 사라진다 (시스템 tips·다른 내 tips는 영향 없음)
- [ ] 새로 고침 → 추가된 내 tips가 그대로 유지된다

### 7. 사례 카드 목록 표시 (페이지 2)
- **Given** — 페이지 2 첫 진입, 체크 데이터 없음
- **When** — 페이지 2를 연다
- **Then** — 큐레이션된 사례 카드 ≥ 8개가 보이며, 각 카드에 제목·설명·외부 링크 표시·미체크 체크박스가 있고, 상단에 "활용한 사례 0 / N" 카운터가 보인다

성공 기준:
- [ ] 페이지 2 진입 → 사례 카드 8개 이상 표시
- [ ] 각 카드 → 제목, 설명, 외부 링크 표시(아이콘 또는 도메인), 체크박스가 모두 보인다
- [ ] 카드의 링크 영역(또는 명시 버튼) 클릭 → 외부 URL이 새 탭에서 열린다
- [ ] 상단 카운터 → "활용한 사례 0 / 8" (전체 시드 수와 일치)

### 8. 사례 체크 → 카운터 갱신·복원
- **Given** — 페이지 2, 8개 카드, 모두 미체크 상태
- **When** — 카드 3개에 체크
- **Then** — 카운터가 "3 / 8"로 갱신되고, 새로 고침해도 체크 상태와 카운터가 유지된다

성공 기준:
- [ ] 3개 카드 체크 → 카운터 "3 / 8"
- [ ] 1개 체크 해제 → 카운터 "2 / 8"
- [ ] 새로 고침 → 체크된 카드만 체크 상태로 보이고, 카운터는 유지된다

### 9. 직군 선택 → 다음 시도 추천 (페이지 3)
- **Given** — 페이지 3 첫 진입, 직군 미선택
- **When** — 직군 select에서 "개발자" 선택
- **Then** — "다음 주 시도할 한 가지" 카드 영역에 개발자 시드 풀에서 1건이 표시된다. 직군을 변경하면 카드 내용이 새 직군의 시드로 교체된다

성공 기준:
- [ ] 직군 미선택 → "직군을 선택하세요" 안내가 보이고, 추천 카드 영역은 비어 있다
- [ ] "개발자" 선택 → 개발자 시드 풀에 속한 시도 1건이 카드(제목 + 설명)로 보인다
- [ ] "디자이너"로 변경 → 디자이너 시드 풀의 시도 1건이 카드에 보인다 (개발자 시드는 더 이상 보이지 않음)

### 10. 이번 주 시도 입력 (1–3줄, 최대 5)
- **Given** — 페이지 3, 직군 선택 완료, 이번 주 시도 영역 빈 상태
- **When** — "Cursor로 React 컴포넌트 1건 작성" 입력 후 추가
- **Then** — "이번 주 시도" 영역에 해당 텍스트가 보이고 같은 주 안에서 최대 5개까지 추가 가능. 6번째 추가 시도 시 한도 안내가 표시된다

성공 기준:
- [ ] 1번째 추가 → "이번 주 시도" 영역에 텍스트 1개가 보인다
- [ ] 5번째까지 정상 추가
- [ ] 6번째 추가 시도 → "이번 주는 최대 5개까지 입력할 수 있습니다" 안내, 추가되지 않음
- [ ] 같은 주 항목 → 수정·삭제 가능
- [ ] 새로 고침 → 입력된 시도 모두 유지

### 11. 회고 뷰 — 최근 4주 표시, 지난 주 read-only
- **Given** — 페이지 3, 지난 3주 동안 매주 시도가 1–2개 기록되어 있음
- **When** — 페이지 3의 회고 영역을 본다
- **Then** — 최근 4주(이번 주 포함)가 주별 그룹으로 보이고, 지난 주 항목은 시각적으로 read-only(회색 처리, 수정/삭제 버튼 없음)임이 확인된다

성공 기준:
- [ ] 회고 영역 → "이번 주", "지난 주", "2주 전", "3주 전" 4개 그룹이 보인다
- [ ] 지난 주 그룹의 항목 → 수정 버튼·삭제 버튼이 보이지 않는다 (read-only)
- [ ] 이번 주 그룹의 항목만 → 수정·삭제 버튼이 보인다
- [ ] 5주 전 시도는 회고 뷰에 보이지 않는다

### 12. 직군 시드 풀 소진 처리
- **Given** — 직군 시드 풀의 모든 시도가 최근 4주 안에 이미 추천되었던 상태
- **When** — 페이지 3에서 같은 직군의 추천 카드를 본다
- **Then** — 카드 자리에 "현재 직군의 추천 풀을 모두 보셨습니다. 다른 직군을 시도해 보세요" 안내가 보인다

성공 기준:
- [ ] 시드 모두 소진 직군 → 안내 메시지 표시, 카드 영역엔 시드가 나타나지 않는다
- [ ] 다른 직군 선택 → 그 직군의 시드 풀에서 1건이 정상 표시된다

### 13. 페이지 간 네비게이션
- **Given** — 페이지 1에 데이터 입력 후
- **When** — 네비게이션을 통해 페이지 2 → 페이지 3 → 페이지 1로 이동
- **Then** — 각 페이지가 자기 데이터를 그대로 표시하며, 페이지 1로 돌아왔을 때 입력값이 유실되지 않는다

성공 기준:
- [ ] 페이지 1에서 활용율·todo·tips 입력 → 페이지 2로 이동 → 페이지 1로 복귀 시 모든 값 유지
- [ ] 페이지 2에서 체크 → 페이지 3으로 이동 → 페이지 2로 복귀 시 체크 상태 유지
- [ ] 페이지 3에서 직군·시도 입력 → 다른 페이지로 이동 → 페이지 3 복귀 시 모든 값 유지

## 불변 규칙

- **데이터 영속성** — 모든 입력값(활용율, 3종 todo, 시스템·내 tips, 사례 체크 상태, 직군, 이번 주 시도, 회고)은 같은 브라우저에서 재방문 후에도 자동 복원된다
- **활용율 입력 범위** — 모든 활용율 입력 지점에서 0 이상 100 이하 정수만 저장된다. 범위 밖은 저장되지 않고 안내가 표시된다
- **반응형 레이아웃** — 360px (모바일) ~ 1280px+ (데스크톱) 폭에서 모든 페이지의 핵심 입력·표시·네비게이션이 가능하다
- **외부 링크** — 모든 외부 링크는 새 탭에서 열린다. 같은 탭 이동을 야기해서는 안 된다
- **추천 비반복** — 페이지 3의 다음 시도 추천은 같은 직군에서 같은 시도를 4주 안에 반복 제시하지 않는다
- **주 경계** — "이번 주 / 지난 주" 구분은 월요일 00:00 ~ 일요일 23:59 (사용자 로컬 시각) 기준이다

## 의존성

- **AI 활용 사례 시드 카탈로그** (페이지 2): 제목 + 설명 + 외부 URL ≥ 8건. 작성 주체는 개발자(정적)
- **직군별 시도 시드 풀** (페이지 3): 직군 ≥ 6개, 각 직군당 시도 ≥ 5개. 작성 주체는 개발자(정적)
- **시스템 Tips 카탈로그** (페이지 1): 격차 카테고리(큼/중간/작음)별 tips ≥ 3개씩. 작성 주체는 개발자(정적)
- **직군 목록**: 시도 시드 풀과 동일한 키 셋트(예: 개발자, 디자이너, 마케터, PM, 기획자, 일반사무직 등 ≥ 6)

## 미결정 항목

- **Daily / Weekly / Monthly Todo의 자동 만료** — 하루/한 주/한 달 지난 todo가 자동으로 사라지는가, 누적 보관되는가? 아카이브가 있는가?
- **Function 3의 사례 link 출처** — 정적 큐레이션으로 시작 (현재 spec). 라이브 Google 검색은 검색 API 비용·quota 부담으로 v0에서 제외. 향후 사용자 피드백 보고 v1에서 검색 통합 여부 결정
- **활용율 단위** — 정수 0–100 % 만 허용 (현재 spec). 소수점 허용 여부는 추후 결정
