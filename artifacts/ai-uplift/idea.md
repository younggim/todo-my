# AI Uplift — 매주 한 가지 시도

## Problem Statement
HMW — 사람들이 임의의 '활용율 %'를 채우려 하지 않고, 다음에 시도할 한 가지 AI 사용법을 매주 실제로 행동에 옮기게 만들 수 있을까?

## Recommended Direction
**"이번 주 시도 로그 + 다음 시도 한 가지"** 단일 루프에 집중한다.

- 사용자는 매주 "새로 시도한 AI 사용법 1–3줄"을 기록한다.
- 시스템은 직군 키워드를 한 번 받아 "다음 주 시도할 한 가지"를 제시한다.
- 활용율 %, 일/주/월 todo, 사례 검색·체크박스 — 모두 v0에서 제외.

이유: 원안의 6개 기능이 동시에 동작하지 않으면 의미가 깨지지만, 이 단일 루프는 한 번 도는 것만으로 가치가 입증된다. 가장 위험한 가정(사용자가 기록 행동을 지속한다)을 가장 적은 빌드로 친다.

## Key Assumptions to Validate
- [ ] 사용자가 한 주에 1번 시도 기록을 입력한다 — MVP 출시 후 4주간 retention 측정 (W2 ≥ 50%)
- [ ] "다음 시도 추천" 한 줄이 실제 시도로 이어진다 — 다음 주 입력 1줄이 추천과 의미적으로 일치하는 비율 ≥ 30%
- [ ] 직군 키워드 기반 정적 큐레이션이 최소 품질 임계를 넘는다 — 추천에 대한 thumbs-up/down ≥ 60% positive

## MVP Scope

**포함**:
- 직군 1회 입력 (자유 텍스트)
- 이번 주 시도 1–3줄 입력
- "다음 주 시도할 한 가지" 표시 (직군 → 정적 추천 매핑, 수동 큐레이션 시드)
- 지난 주 시도 로그 회고 뷰

**제외**:
- 활용율 % 입력·표시
- 일/주/월 todo 3종
- Google 사례 검색·link·체크박스
- LLM 호출
- 백엔드·인증 (LocalStorage 우선)

## Not Doing (and Why)
- AI 활용율 % 입력 — 측정 대상이 모호한 vanity metric, 행동 변화 인과가 불명
- 일/주/월 todo 3종 — 동기 비용 폭탄, 핵심 루프 외부 기능
- Google 검색 사례 + 체크박스 — 큐레이션 품질이 본 게임이 되어 우선순위 흐림
- LLM 자동 추천 — 정적 큐레이션이 임계 통과 전엔 인프라 비용만 늘림
- 팀/공유 기능 — 1인 모드 가치 검증 후 판단
